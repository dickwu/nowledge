# Security Policy

## Supported versions

Security fixes are made on the default branch and included in the next release.
Older commits and superseded releases are not maintained separately.

## Reporting a vulnerability

Do not open a public issue or discussion. Sign in to GitHub and use the
[private vulnerability reporting form](https://github.com/dickwu/nowledge/security/advisories/new).
That form keeps the report and follow-up discussion private between the reporter
and repository maintainers.

Include the affected version or commit, impact, reproduction steps, and any
known mitigation. Remove live credentials, personal data, and unrelated tenant
data from examples. Maintainers will acknowledge a complete report within seven
days and provide a status update at least every fourteen days until it is closed.

If GitHub does not show the private reporting form, do not disclose the issue in
public; private vulnerability reporting must be enabled by a repository
administrator before this channel can accept reports.
